# TA Front-End Assessment Rubric (Levels 1-4)

This rubric maps the repository directly to Levels 1-4.

## Level Coverage

| Level | Problems | Technical focus | TA communication focus |
| --- | --- | --- | --- |
| Level 1 | `problem1.html`, `problem2.html` | HTML structure, CSS basics, JavaScript/DOM basics | Explains simple fixes clearly and correctly |
| Level 2 | `problem3.html`, `problem4.html` | Bootstrap grid, responsive layout rules | Explains layout reasoning in beginner-friendly language |
| Level 3 | `problem5.html`, `problem6.html` | Events/forms, async `fetch`, loading and error states | Describes why bug fixes work, not just what changed |
| Level 4 | `problem7.html`, `problem8.html` | Complex logic debugging, data validation, edge-case handling | Teaches the fix as if mentoring a student |

## Scoring (0-3 per problem)

- `0`: Not functional or mostly incomplete.
- `1`: Partial solution with major logic/syntax issues left.
- `2`: Functional solution with minor issues (edge cases, clarity, or robustness).
- `3`: Correct, readable, and fully meets prompt requirements.

## Level Pass Thresholds

- Level 1 pass: average `>= 2` across Level 1 problems.
- Level 2 pass: average `>= 2` across Level 2 problems.
- Level 3 pass: average `>= 2` across Level 3 problems.
- Level 4 pass: average `>= 2` across Level 4 problems and no score of `0`.

## Interview Notes for TA Hiring

- Ask the candidate to explain one fix from each completed level.
- Prioritize root-cause explanations over memorized syntax.
- Favor code that a beginner could read and maintain.
- At Level 4, require at least one teaching-style explanation that is concise and accurate.
